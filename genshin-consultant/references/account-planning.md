# Account Planning

Use this reference when the consultation goes beyond one character and asks for party, roster, pull, weapon allocation, or long-term account direction.

## Planning order

1. Identify the user's goal: story/field comfort, Abyss, Theater, high-difficulty bosses, favorite-character investment, or low-spend efficiency.
2. Stabilize one main team before optimizing the whole account.
3. Build a second team only after the first team has working ER, survival, and correct artifact main stats.
4. For Theater-style breadth, widen the roster after the main teams are functional.
5. Treat pull advice as optional and cost-sensitive.

## Account diagnosis categories

- **Immediate bottleneck**: the one issue that most blocks the current goal, such as ER, survivability, missing Hydro/Dendro applicator, wrong artifact main stat, misunderstood kit mechanic, or underleveled weapon.
- **Role gaps**: healer, shielder, off-field Hydro, off-field Dendro, Anemo VV holder, buffer, battery, bow user, shield breaker, reaction trigger, interruption resistance.
- **Resource bottleneck**: resin, mora, EXP books, talent books, weekly boss drops, weapon materials, artifact fodder, billets, fragile resin, primogems.
- **Team bottleneck**: two teams sharing the same support, no defensive unit, reaction ownership conflict, burst cooldown mismatch, or one weapon needed by multiple characters.

## Three-team recommendation rule

When the user provides owned characters, recommend exactly 3 teams unless the roster cannot support 3 coherent options.

Default categories:

1. **현재 최우선 팀** — strongest practical team for the user's main carry or goal.
2. **안정/저투자 팀** — easier rotation, defensive comfort, lower artifact pressure.
3. **2파티/장기 성장 팀** — Abyss second side, Theater breadth, or future farming direction.

For each team include:

- four members and replacements if the roster is incomplete
- each member's role
- why the team works
- basic rotation sketch
- minimum ER or stat conditions
- weapon conflicts and artifact domain conflicts
- what character/material to build first

Rules:

- Do not put the same character into two simultaneous Abyss teams unless the row is labeled `대체안`.
- Include a defensive option when practical for new players.
- If a recommended team requires an unowned character, label it as `future upgrade`, not current advice.
- If the roster lacks core enablers, give temporary teams plus the next role to acquire.
- If weapon inventory is available, assign weapons per team and flag conflicts.

## Pull advice rules

- Never frame a pull as mandatory.
- Explain opportunity cost: primogems spent now reduce future guarantee/favorites.
- Prefer account-enabling roles over luxury constellations for new players.
- Separate `account value`, `favorite/comfort value`, and `meta/endgame value`.
- Give alternatives using owned characters or 4-star options when possible.
- If current banner, future kit, or rerun timing is time-sensitive, browse current sources before advising.

## Output shape for account plans

Use three time horizons:

- **Today**: actions possible without waiting for weekly resets or major farming.
- **This week**: resin plan, boss/material schedule, artifact domain focus.
- **Long term**: second team, pull priorities, Theater breadth, luxury optimization.

For each recommendation, include:

- priority number
- target character/team
- required action
- reason
- source/evidence or uncertainty label

## Account visual outputs

When recommending parties, create or describe a `party_card` for the primary recommendation and optionally for the other two teams if the user asked for visuals. The card should line up four characters in one row, label roles, and point to the same source IDs used for the text explanation. If roster images or safe official/wiki portraits are unavailable, use placeholders and keep the metadata explicit.
